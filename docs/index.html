How it works

Three steps. Every AI knows you.

1

Scan your machine

aura detects your languages, frameworks, tools, projects, and editor automatically. Zero YAML writing required.

aura scan # or the full flow: aura quickstart
2

Answer 5 questions

Your role, communication style, current priorities, and rules. 2 minutes to capture what scanning can't detect.

aura onboard # or add facts directly: aura add dev tone "Direct"
3

Serve via MCP

Start the built-in MCP server. Claude Desktop, Cursor, and any MCP client read your context automatically.

aura setup # config tools aura serve # start MCP
Context Packs

Your identity, structured

Context packs are human-readable YAML. Version them, share them, edit them.

~/.aura/packs/developer.yaml
name: developer scope: development facts: - key: languages.primary value: [TypeScript, Python] confidence: high - key: frameworks value: [Next.js 15, FastAPI, Supabase] - key: editor value: Cursor rules: - instruction: "Always use TypeScript strict mode" priority: 8 - instruction: "Prefer functional patterns over OOP" priority: 5

MCP Server

The real power:
built-in MCP

aura includes a full MCP server. Start it once and every compatible tool — Claude Desktop, Cursor, ChatGPT Desktop — reads your context automatically.

No copy-paste. No re-explaining. Your context packs become live resources that AI tools query in real time.

Works with any tool that supports the Model Context Protocol.

claude_desktop_config.json
{
  "mcpServers": {
    "aura": {
      "command": "npx",
      "args": ["mcp-remote",
        "http://localhost:3847/mcp"]
    }
  }
}
terminal
$ aura setup  # auto-config Claude + Cursor
